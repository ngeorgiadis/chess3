# Chess Mentor Studio

A local-first Electron desktop app for chess improvement, implemented from the spec pack in
[`chess_training_app_specs/`](chess_training_app_specs/README.md). Import your games, analyze mistakes with
a local UCI engine, practice openings with spaced repetition, and study validated interactive lessons —
optionally generated by AI from material you are authorized to use.

## Quick start

```bash
npm install
npm run dev        # development with hot reload
```

or build and run:

```bash
npm run build
npx electron .
```

Self-test (no network or engine needed):

```bash
npm test           # builds, then runs the app with --smoke-test against a temp database
```

## Using the app

1. **Settings** — enter your Chess.com/Lichess username so imports detect which side you played.
2. **Import games** — sidebar button; username, URL, pasted PGN, or a `.pgn` file.
3. **Engines** — add a UCI engine executable (e.g. [Stockfish](https://stockfishchess.org/download/)).
   The app warns before running any executable and verifies it with a UCI handshake.
4. **Games → Analyze** — queues engine analysis; mistakes become review cards and exercises.
5. **Today** — follow the generated plan: exercises due, opening lines due, games to review.
6. **AI Studio** — optional. Configure an OpenAI-compatible or local endpoint in Settings, then
   paste authorized notes/text and generate a schema-validated lesson. Manual JSON mode works without AI.

## Architecture

- **Electron main process**: SQLite (`node:sqlite`, zero native deps), importers, UCI child processes,
  persistent job queue, Ajv validation, AI provider calls. See [src/main](src/main).
- **Preload**: minimal typed `contextBridge` API, no raw `ipcRenderer` exposure.
- **Renderer**: React 18 + TypeScript + Zustand, custom SVG/CSS chessboard (no proprietary assets),
  strict CSP, sandboxed, no Node access. See [src/renderer](src/renderer).
- **Shared**: types and JSON schemas in [src/shared](src/shared).

Data lives in the OS user-data directory (`%APPDATA%/chess-mentor-studio` on Windows):
`app.db` plus `engines/ backups/ exports/ lessons/ logs/`.

## Security & ethics notes (per spec)

- `contextIsolation: true`, `sandbox: true`, `nodeIntegration: false`, strict CSP, allowlisted import hosts.
- Ongoing games are flagged and blocked from engine analysis — no live-game assistance.
- Engines are user-selected local binaries, spawned with argv arrays (never a shell), killed on timeout.
- Nothing is sent to an AI provider unless you explicitly click generate; local-only mode is the default.
- The AI lesson agent refuses long verbatim reproduction and records a rights mode on every source.
